C Language


22 May '24

I first encountered the C programming language during my undergraduate days as part of my introductory course to computer science. Back then, it was a challenge for me as programming was new to me, with no prior knowledge whatsoever. Address, pointers and memory allocations were enough to throw me off programming, wondering if I will ever make it in programming. I recently revisited C in order to build more breadth in my programming skills. I have been using Java in my new role and I thought that it would be nice to supplement my skills with some C/C++ knowledge.

C was created in the 1970s by Dennis Ritchie. It is an imperative procedural language with uses in operating systems and device drivers. C does not have object-orientation and garbage collection. Many languages have been built on the ideas of C, such as C++ and Java. Thus when I read C's syntax, I found it strikingly similar to Java.

Coming back to my revisiting of the C programming language, I dived straight into the topics of pointers, address and memory management as these are not taught in Java. Java handles pointers, addresses, and memory management under the hood by abstracting these details from the programmer, providing a safer and more managed environment. Let's discuss pointers, addresses and memory management briefly. Pointers and addresses will be discussed together their usage in C language depends on each other. Memory management will have its own topic.

Pointers and Address

Pointers are variables that store the memory address of another variable. There are a few ways to declare a pointer. a) int* p; b) int *p; c) int * p; int* p; int *p; int * p; All 3 approaches will work fine and it depends on individual preference on which approach to use. Personally, I prefer approach a). Visually, it tells me that p is a pointer which points at a memory that stores an integer. However, be mindful that in more complex declarations, such as `int* p, q;`, it might not be clear that `q` is not a pointer. For this reason, some prefer the style `int *p;` to avoid confusion when declaring multiple variables in the same statement. Pitfall: int* p, q; misinterpret
